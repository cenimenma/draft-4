### 2. GitHub Actions 配置

#### `.github/workflows/auto-review.yml`
- **触发条件**：PR 创建、更新、重新打开
- **权限**：pull-requests: write, contents: read
- **步骤**：
  1. Checkout 代码
  2. 设置 Python 环境
  3. 安装依赖（requests）
  4. 获取变更文件
  5. 运行 auto_review.py
  6. 上传评审结果

**关键配置**：
```yaml
on:
  pull_request:
    types: [opened, synchronize, reopened]
    branches: [main, develop]

permissions:
  pull-requests: write
  contents: read
```

---

### 3. 自动评审脚本

#### `auto_review.py` - 完整版
- **类**：`ArgusAutoReviewer`
- **方法**：
  - `get_changed_files()` - 获取变更文件
  - `review_file()` - 调用 API 评审
  - `format_comment()` - 格式化评论
  - `post_pr_comment()` - 发布到 GitHub PR
  - `run()` - 主流程

**工作流程**：
```
1. 检测变更文件
   ↓
2. 逐个文件提交 API 评审
   ↓
3. 格式化评审结果为 Markdown
   ↓
4. 通过 GitHub API 发布评论
```

## 📊 数据流

### 输入 → 处理 → 输出

```
INPUT:
  - Git diff (code changes)
  - File path
  - Project name

PROCESSING:
  1. Parse diff
  2. Build prompt:
     "You are a Linux kernel code review expert..."
  3. Call LLM (Qwen2.5-Coder-1.5B)
  4. Parse response

OUTPUT:
  - List of issues
  - Severity levels (Critical/Warning/Info)
  - Suggestions
  - Formatted Markdown comment
```
