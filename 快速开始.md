# 🚀 Argus Auto Review - 极限冲刺版

## ⚡ 两步搞定 AI 自动代码审查

### 🛠️ 第一步：配置 GitHub Actions（已完成✅）

文件位置：`.github/workflows/ai_review.yml`

**作用**：只要你 push 代码或提 PR，GitHub 会自动开一台临时服务器，把变动的代码抓下来，然后丢给 Python 脚本。

---

### 🧠 第二步：AI 审查脚本（已完成✅）

文件位置：`ai_review.py`

**功能**：
1. 自动抓取本次 Push 的代码变化（git diff）
2. 拼成 Prompt 丢给大模型（DeepSeek/OpenAI）
3. 自动把模型意见以评论方式贴回 GitHub！

---

## 🎯 演示步骤（开会前必看）

### 1️⃣ 配置 API Key

进入你的 GitHub 仓库：
```
Settings → Secrets and variables → Actions → New repository secret
```

添加密钥：
- **Name**: `LLM_API_KEY`
- **Value**: 你的 DeepSeek/OpenAI API Key

> 💡 如果没有 API Key，可以用免费的 [DeepSeek](https://platform.deepseek.com/) 注册获取

---

### 2️⃣ 故意写一段有 Bug 的代码

例如，在项目里新建一个测试文件 `test_bug.py`：

```python
# test_bug.py
def buggy_function():
    # 死循环
    while True:
        pass
    
    # 未定义变量
    print(undefined_variable)
    
    # 潜在的空指针
    x = None
    x.do_something()
```

---

### 3️⃣ 提交并推送

```bash
git add .
git commit -m "test ai review"
git push
```

---

### 4️⃣ 见证奇迹时刻！✨

稍等 **30 秒 - 1 分钟**，刷新 GitHub 对应的 Commit 页面：

你会看到在你的代码下方，已经自动挂上了 AI Agent 生成的、整整齐齐的 Markdown 审查报告！

例如：

```markdown
### 🤖 AI Agent 自动化代码审查报告

#### 🔴 严重问题

1. **死循环风险**

   - 建议：使用前检查 `x is not None`

#### 💡 改进建议
...
```

## 🐛 常见问题

### Q: Workflow 没有触发？
A: 检查：
1. Actions 标签页是否启用了 workflows
2. 是否推送到 `main` 或 `master` 分支
3. 查看 Actions 日志是否有错误

### Q: API 调用失败？
A: 确保：
1. `LLM_API_KEY` secret 已正确设置
2. API Key 有效且有余额
3. 查看 Actions 日志中的错误信息

### Q: 评论没有发布？
A: 检查：
1. Workflow 是否有 `pull-requests: write` 权限
2. GITHUB_TOKEN 是否有效
3. Commit 是否在正确的分支上
