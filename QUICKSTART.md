# 🚀 Argus Auto Review - 5分钟快速上手

## 🎯 目标

用 **GitHub Actions + Python 脚本** 实现 AI 自动评论代码，打通整条运维流程。

---

## ⚡ 3步快速演示

### Step 1: 运行演示（立即看到效果）

```bash
cd draft_4
run_demo.bat
```

这会模拟完整的自动化流程：
- ✅ 检测代码变更
- ✅ 调用 AI 评审 API
- ✅ 生成 GitHub PR 评论

**预期输出：**

### Step 2: 部署到 GitHub（生产环境）

#### 2.1 推送代码

```bash
git init
git add .
git commit -m "Add auto-review workflow"
git remote add origin https://github.com/YOUR_USERNAME/YOUR_REPO.git
git push -u origin main
```

#### 2.2 配置 GitHub Secrets

1. 进入 GitHub 仓库
2. 点击 **Settings → Secrets and variables → Actions**
3. 添加 Secret：
   ```
   Name: ARGUS_API_URL
   Value: http://your-api-server.com  (或 ngrok URL)
   ```

#### 2.3 启用 Workflow

Workflow 已配置在 `.github/workflows/auto-review.yml`，会自动触发。

---

### Step 3: 创建 PR 测试

1. 创建一个新分支
2. 修改一些代码
3. 提交并创建 Pull Request
4. **等待 1-2 分钟**
5. 查看 PR 评论区 - AI 会自动发表评论！🎉

## 💡 关键配置

### GitHub Actions 权限

在 `auto-review.yml` 中已配置：
```yaml
permissions:
  pull-requests: write  # 允许发表评论
  contents: read        # 允许读取代码
```

### 环境变量

| 变量 | 来源 | 说明 |
|------|------|------|
| `GITHUB_TOKEN` | 自动提供 | GitHub 访问令牌 |
| `GITHUB_REPOSITORY` | 自动提供 | 仓库名称 |
| `PR_NUMBER` | 自动提供 | PR 编号 |
| `ARGUS_API_URL` | GitHub Secrets | API 地址 |

## 🐛 常见问题

### Q: Workflow 没有触发？
A: 检查：
1. Actions 标签页是否启用了 workflows
2. PR 是否指向 `main` 或 `develop` 分支
3. 查看 Actions 日志是否有错误

### Q: API 连接失败？
A: 确保：
1. `ARGUS_API_URL` secret 已正确设置
2. API 服务器正在运行且可公开访问
3. 使用 ngrok 测试：`ngrok http 8000`

### Q: 评论没有发布？
A: 检查：
1. Workflow 日志中的错误信息
2. `GITHUB_TOKEN` 权限是否正确
3. PR 是否仍然处于打开状态
