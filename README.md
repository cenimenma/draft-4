# 🤖 Argus Auto Review - GitHub Actions Integration

## 📋 Overview

This is an **ultra-lightweight DevOps version** that automatically reviews code changes using AI and posts comments on GitHub Pull Requests.

**Architecture:**
```
Developer pushes code → GitHub Actions triggers → Calls Argus API → AI reviews → Auto-comments on PR
```

### Option 2: GitHub Actions (Production)

#### Prerequisites
1. Push this code to GitHub
2. Deploy Argus API to a public server (or use ngrok for testing)
3. Configure GitHub Secrets

#### Setup Steps

**Step 1: Deploy Argus API**

You need a publicly accessible API endpoint. Options:
- Cloud server (AWS, GCP, Azure)
- Railway/Render (free tier)
- ngrok tunnel (for testing)

**Step 2: Configure GitHub Secrets**

Go to: `Settings → Secrets and variables → Actions`

Add these secrets:
```
ARGUS_API_URL=https://your-api-domain.com
```

**Step 3: Enable Workflow**

The workflow is already configured in `.github/workflows/auto-review.yml`

It will automatically trigger on:
- ✅ New Pull Requests
- ✅ PR updates (new commits)
- ✅ PR reopens

## 🔧 How It Works

### 1. Trigger
When a developer creates or updates a PR, GitHub Actions automatically starts.

### 2. Extract Changes
The workflow identifies which files were changed and extracts the diff.

### 3. AI Review
Each changed file is sent to the Argus API for AI-powered code review.

### 4. Post Comment
Review results are formatted and posted as a comment on the PR.

## ⚙️ Configuration

### Environment Variables

| Variable | Description | Required |
|----------|-------------|----------|
| `GITHUB_TOKEN` | Auto-provided by GitHub | ✅ Yes |
| `GITHUB_REPOSITORY` | Auto-provided (e.g., "user/repo") | ✅ Yes |
| `PR_NUMBER` | Auto-provided from event | ✅ Yes |
| `ARGUS_API_URL` | Your Argus API endpoint | ✅ Yes |

### Workflow Triggers

Currently configured to trigger on:
```yaml
on:
  pull_request:
    types: [opened, synchronize, reopened]
    branches: [main, develop]
```

You can customize this in `.github/workflows/auto-review.yml`

## 🐛 Troubleshooting

### Issue: Workflow doesn't trigger

**Solution:**
1. Check if workflow is enabled: `Actions` tab → Enable workflows
2. Verify trigger conditions match your PR
3. Check workflow file syntax

### Issue: API connection failed

**Solution:**
1. Verify `ARGUS_API_URL` secret is set correctly
2. Ensure API server is running and accessible
3. Check firewall/network settings

### Issue: Comment not posted

**Solution:**
1. Verify `GITHUB_TOKEN` permissions (pull-requests: write)
2. Check workflow logs for errors
3. Ensure PR is still open
